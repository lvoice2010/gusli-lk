# CLAUDE.md — gusli-lk

Личный кабинет клиента **КиберГусли** (CyberGusli): отчётный дашборд по работе голосового AI-ассистента. Прототип.

## Назначение
ЛК для действующих оплативших клиентов: статистика работы ассистента — сколько обращений, сколько ИИ закрыл сам vs передал оператору (и почему), о чём звонят, сколько это экономит, что ассистент сделал в CRM клиента. Подробное ТЗ — в `Задание_Claude_ЛК_клиентов.md`.

## Стек
- Next.js 14 (App Router) + TypeScript
- Tailwind CSS, тёмная тема КиберГусли (фиолетовый → синий → циан, мотив звуковой волны)
- npm-пакет: `cybergusli-lk`

## Структура
- `app/(lk)/` — разделы ЛК: `home`, `dashboard`, `assistants`, `reports`, `documents`, `messages`, `catalog` (Витрина услуг), общий `layout.tsx`
- `app/page.tsx` — редирект на `/dashboard`
- `app/globals.css` — стили и палитра

## Команды
```bash
npm install
npm run dev      # http://localhost:3000 → редирект на /dashboard
npm run build
npm run start
npm run lint
```

## Конвенции / важное
- Боковое меню по образцу O'LINE: Главная · Ассистенты, группа Управление (Фин. документы · Отчёты · Отправить сообщение), выделенная цветом Витрина услуг.
- Переключатель тёмная/светлая тема внизу сайдбара, состояние в `localStorage`.
- Оплата по факту (без пакета минут).
- Сохранять текущий стек, стиль и компоненты графиков — переиспользовать, не вводить новые библиотеки графиков без причины.
- Прототип на мок-данных; при интеграции данные по звонкам тянутся по образцу EngineGusli (`ai_admin2`).
