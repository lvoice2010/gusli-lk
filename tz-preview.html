Техническое задание
Личный кабинет клиента «КиберГусли»

Документ описывает логику отображения клиентского ЛК и какие данные он ожидает от систем заказчика. Настройку передачи данных (EngineGusli/ai_admin2, CRM, биллинг, ЭДО) разработчик реализует на своей стороне — здесь только что и в какой логике выводить, за какие периоды, по каким формулам, и что приходит из админского/менеджерского кабинета.

Ресурсы и доступы:

  • Прототип (демо на Vercel): https://gusli-lk.vercel.app
  • Исходный код (Git-репозиторий): https://github.com/lvoice2010/gusli-lk — доступ на клонирование выдаётся команде разработки.
  • Версия: прототип на мок-данных (фронтенд на Next.js; данные демонстрационные).

0. Базовые понятия и принципы

Два контура данных:

  1. Статистика — звонки, диалоги, темы, эскалации, длительности, действия в CRM. Числа, которые система заказчика передаёт за период.
  2. Справочные/настроечные данные — задаются в админском/менеджерском кабинете и отображаются клиенту: договор, контрагент, менеджер, тариф, фин. документы, наполнение Витрины, рекомендации, ставки.

Роли кабинетов:

  • Клиентский ЛК — только просмотр, кроме отправки заявок и сообщений.
  • Менеджерский/админский — источник справочных данных и настроек. Типы: админский, партнёрский, самостоятельный, интеграторский.

Форматы: деньги ₽ (80 680 ₽); длительность мин:сек; доли — проценты (эскалации — с 1 знаком); часовой пояс заведения. Темы: тёмная (по умолч.) и светлая, сайдбар тёмный в обеих.

1. Периоды и сравнения (сквозная логика)

Пресеты: Сегодня · Вчера · Неделя · Месяц · Период (календарь).

  • Месяц = текущий месяц на тек. дату (MTD), не полный (с 1-го по сегодня).
  • Произвольный период — «с»/«по» (не позже сегодня).

Сравнение (дельты ↑/↓) — с аналогичным предыдущим периодом той же длины; бэкенд отдаёт пару значений.

ПериодС чем сравнивается
Сегодняк этому же времени вчера
Неделяк предыдущей неделе
Месяц (MTD)к тому же периоду прошлого месяца (1..N число)
Периодк предыдущему отрезку той же длины

Цвет дельты: для «хороших» рост = зелёный (обращения, закрыто ИИ, конверсия, экономия), для «плохих» рост = красный (эскалации, доля эскалаций, средняя длительность). Флаг higherIsBetter у каждой метрики.

2. Сайдбар

  • Логотип; Главная; Ассистенты (список, статусы; SMS-рассылки скрыты флагом); Управление (Фин. документы, Отчёты, Сообщение); Витрина → /catalog.
  • Внизу: переключатель темы; карточка профиля (ФИО + компания).
  • Бейдж «Sk Участник» — ссылка на сайт Сколково (sk.ru), новая вкладка.

Из админки: список ассистентов и статусы, ФИО контакта, компания, видимость пунктов меню.

3. Шапка

  • Заголовок раздела, «назад» где есть.
  • Чип «к оплате / задолженность»: при долге — янтарный с минусом и ⚠ («−80 680 ₽»). Значение = текущая сумма к оплате (§6.2).
  • На мобиле слева — бургер (открывает сайдбар-шторку).

4. Главная (/home)

  1. Рекомендации — карточки услуг, отмеченные менеджером (§9). Cta → форма заявки (§8); «Демо» → демо-окно.
  2. Каталог — баннер-ссылка на /catalog.
  3. Подключённые услуги — карточки ассистентов: статус, метрики (Закрыто ИИ %, Эскалации %, ср. длит.), принято за месяц, дата, менеджер, «Подробнее» → дашборд.

5. Дашборд услуги (/dashboard)

Шапка услуги: название, статус, номер линии, блок «Тест ассистента» (звонок в браузере, WebRTC), блок «Менеджер» (из справочника, §7), блок «Отчёты». Раскрывающийся «Условия и состав услуги» (по умолч. свёрнут): № договора, контрагент, график, состав — из админки. На мобиле три блока внутри «Условия и состав». Вкладки: Отчёты · База знаний · Диалоги («Аналитика» скрыта).

5.1. Отчёты

Переключатель периода (§1); по умолчанию Месяц (MTD).

KPI (5 карточек)

  • Всего обращений = total.
  • Закрыто ИИ = aiClosed; % = aiClosed/total.
  • Эскалации = escalated; % = escalated/total (меньше — лучше).
  • Средняя длительность = avgDurationSec (меньше — лучше).
  • Минут за период = суммарная длительность обработанных диалогов.

Блоки

  • ИИ vs оператор + причины перевода (количество и доля от эскалаций; из систем, сумма = escalated).
  • Тематика обращений (топ-тем + дельта).
  • Все обращения (Целевые = total − nontarget).
  • Конверсия в заявки = заявки/целевые.
  • Почасовая карта 7×24, селектор Неделя/Месяц/Квартал.
  • Месяц к месяцу: «Закрытые» (два завершённых) и «Текущий MTD» (тек. месяц vs те же даты прошлого, §6.4).
  • Помесячная динамика (12 мес).
  • Годовой отчёт по месяцам; текущий месяц зелёный «на тек. дату».
  • Статус интеграций (действия за период + лента); селектор Месяц/Квартал/Год.
  • ROI / экономия — селектор Месяц/Квартал/Год; формула §6.3.

5.2. База знаний

Разделы-аккордеоны (по умолчанию все свёрнуты), цветные шапки, внутри — сплошной текст (поддержка **жирного** и списков), источник и дата. CTA «Написать менеджеру». Контент — из админки.

5.3. Диалоги

Фильтры: период, сегмент Все / Закрыто ИИ / Эскалации, поиск. Пагинация по 20. Колонки: Время, Номер, Тема, Тип, Эскалация, Длит., Результат, «Открыть».

Карточка вызова (окно по центру), 3 колонки:

  • Информация: UID, номер звонящего, номер назначения, линия, Ассистент (не оператор), Ожидание ответа «менее 1 сек», длительность, тема. Холда нет.
  • ИИ-карточка: ФИО, phone_number, суть обращения, категория, подкатегория.
  • Стенограмма с таймкодами.
  • Запись: плеер + «Скачать запись» + «Скопировать ссылку».

6. Формулы

6.1. Проценты

  • Закрыто ИИ % = aiClosed / total × 100
  • Эскалации % = escalated / total × 100
  • Целевые = total − nontarget; Конверсия = заявки / целевые × 100

6.2. К оплате

к оплате = минуты_за_месяц × цена_минуты + стоимость_сопровождения

цена_минуты и стоимость_сопровождения — из админки (прототип: 13,5 ₽/мин, 14 800 ₽/мес).

6.3. ROI / экономия (с учётом занятости оператора)

  • разговорные_минуты = Σ(aiClosed × avgDialogSec) / 60
  • занятость (occupancy) — из админки, по умолч. 0,7
  • сэконом._оплач._минуты = разговорные_минуты / занятость
  • экономия_₽ = (сэконом._минуты / 60) × ставка_оператора_в_час (по умолч. 350 ₽/час)
  • стоимость_диалога_ИИ = (avgDurationSec / 60) × цена_минуты

Оператор не загружен на 100% → для покрытия N разговорных минут нужно больше оплачиваемого времени.

6.4. MTD-сравнение

  • доля_дней = прошедших_дней / дней_в_месяце
  • объёмные метрики прошлого месяца × доля_дней (ставки/средние не масштабируются)

8. Заявки и сообщения (куда уходят)

Клиент не редактирует данные ЛК, только отправляет обращения. У каждого — свой канал (e-mail):

  • «Подключить услугу» (Витрина и «Оставить заявку» на главной): услуга, комментарий, срочность → письмо на sales@cybergusli.ru. Подтверждение «Заявка отправлена».
  • «Написать менеджеру» — письмо на e-mail прикреплённого менеджера (справочник, §7).
  • «Сообщить о неточности» (у записей базы знаний) — письмо менеджеру с привязкой к записи.

Бэкенд: отправка писем (SMTP/почтовый сервис заказчика). В письме: клиент, компания, услуга/раздел, комментарий, срочность, дата, ссылка на ЛК.

9. Рекомендации клиенту (галочки)

В менеджерском кабинете — общий каталог услуг, у каждой услуги чекбокс «рекомендовать этому клиенту» + персональный инсайт. ЛК выводит в «Рекомендуем вам» только отмеченные услуги. Настраивается индивидуально для каждого клиента.

10. Витрина услуг (/catalog)

Каталог — единый общий справочник услуг (как справочник менеджеров): услуга заводится один раз и используется для всех клиентов; персонализация — через рекомендации и видимость, без дублирования карточек.

  • Герой + статистика; секции друг под другом: «Рекомендуем вам» (§9), «Новинки», «Все услуги».
  • Карточка: иконка, бейджи (Новинка / Подходит вашей отрасли / Скоро), название, описание, фичи, инсайт, цена, кнопки «Демо» и «Подключить» (→ §8). «Скоро» — кнопка неактивна.

Из админки (с правом правки): название, описание, фичи, цена, бейджи, инсайт, видимость, «Скоро», демо. Демо (напр. «Ежеквартальный ИИ-отчёт») — пример отчёта на обезличенных данных; при подключении строится на данных клиента.

11. Фин. документы (/documents)

  • Сводка по году: сумма счетов, к оплате, актов ждёт подписи.
  • Фильтры: год, тип (Счёт/Акт/Договор), статус, поиск. Таблица: дата, документ, период, сумма, статус, «Посмотреть» → PDF (окно) + «Скачать».

11.1. Суммы и пересчёт по статусам

  • Сумма счёта за месяц = минуты_месяца × цена_минуты + стоимость_сопровождения (§6.2). По акту — та же сумма за период.
  • Сводка по статусам: «Сумма счетов за год» = Σ всех счетов года; «К оплате» = Σ счетов «К оплате»; «Актов ждёт подписи» = число актов «Ожидает подписи».
  • Чип в шапке = задолженность = счета «К оплате».
  • При смене статуса (К оплате → Оплачен, Ожидает подписи → Подписан) сводка и чип пересчитываются автоматически.

Из менеджерского аккаунта: файлы PDF и метаданные. Статусы оплаты — из биллинга, подписания — из ЭДО; суммы — по формуле (минуты передаёт статистика).

12. Отчёты (/reports) и Сообщения (/messages)

  • Отчёты — архив сформированных отчётов (PDF), просмотр/скачивание.
  • Сообщения — канал связи с менеджером.

ЛК ожидает данные по виджетам в описанной логике и за указанные периоды; передачу данных из систем заказчика (EngineGusli/ai_admin2, CRM, биллинг, ЭДО) заказчик настраивает самостоятельно.

13. Авторизация, клиенты и размещение

13.1. Вход

  • Вход по логину + паролю. Логин — номер телефона ИЛИ e-mail клиента. Пароль — выданный нами.
  • Самостоятельной регистрации нет. Доступы создаём мы в менеджерском кабинете; клиент только входит.
  • Сессия с «запомнить меня»; выход из ЛК. Восстановление пароля — сброс по e-mail/телефону или через менеджера.

13.2. Клиенты в менеджерском кабинете

В менеджерском/админском кабинете — список клиентов. По каждому: контактное лицо (ФИО), название компании (контрагент), логин (телефон и/или e-mail), пароль (выдаём/сбрасываем мы), привязанные ассистенты/услуги, прикреплённый менеджер (§7).

Бэкенд: пароли — в хешированном виде; вход по телефону/e-mail; ограничение попыток. Данные берутся из менеджерского кабинета.

13.3. Размещение

  • Поддомен: ЛК размещается на поддомене lk.cybergusli.ru (на стороне команды разработки/их хостинге) с обязательным SSL (HTTPS).
  • DNS и маршрутизация настраиваются под выбранный командой хостинг; DNS-записи поддомена прописываются у регистратора домена cybergusli.ru.
  • Тема по умолчанию — светлая; тёмная доступна переключателем, выбор сохраняется.
